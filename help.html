What it is

A phone-first workout tracker built so Joe gets back in the groove of training — log fast the way he marked up paper calendars. It's a PWA (installable web app) on free GitHub Pages, syncing across phone + computer via Supabase.

Operating principle (from the panel): personal-first, product-optional. The only KPI in Phase 0 is workouts logged per week. Features earn each escalation with real retention data, and the whole thing sits behind the KatchT go-live gate — KatchT is the bet, GROOVE is founder dogfood.

How it works

  • Grid logging: your exercises are rows with reps/sets/lbs boxes (or distance + hr/min for cardio, hr/min for classes). Fill what you did → Log it.
  • Sets: tap once per set — tally | || ||| then 4×, 5×… (toggle to type a number in Settings).
  • Exercise types: Strength (reps×sets×lbs) · Cardio (distance + hr/min) · Activity (hr/min — Jiu Jitsu, Yoga).
  • Add a move: name + abbreviation + type → joins your grid with the right boxes.
  • Manage moves: rename, retype, delete, and reorder (▲▼) by priority.
  • Text reps (optional): AMRAP / 8-12 / failure; plain numbers still save as numbers.
  • Drafts: in-progress entries survive reordering and app restarts (never lose typed numbers).
  • Settings: light/dark theme · show/hide daily quote · tap-vs-type sets · text reps · version + self-update.

Points engine

Mirrors the Command Center's philosophy: a lifetime total that never resets (trophy room) + a monthly layer that resets the 1st (fresh-start dopamine) + a streak. Auto-derived from real logs — never minted.

  • Showed up +20 · per exercise +3 (cap 15) · volume +1/50 reps (cap 10) · cardio/time bonus (cap 10)
  • New move +5 · 3× in a week +30 · comeback after a break +15
  • Daily streak multiplier: 3d ×1.2 · 7d ×1.5 · 14d ×1.75 · 30d ×2
  • Ranks (monthly): Warming Up → Finding the Groove → In the Groove → Locked In → Relentless

Data model (Supabase)

Dedicated free project workout-tracker (ref ebxyickzytkywkxlcrlf) in a separate "personal" org so it stays free even after KatchT's org goes Pro. Row-Level Security isolates all data by user.

Auth: email + password, signups closed after Joe's account (invite-only). Drafts cached in localStorage. Schema in schema.sql.

Roadmap

Phase 0 — Dogfood Now

Use it daily for 4–6 weeks. Fix only what annoys mid-workout. No store, no marketing, no big features.

Gate → advance only if: Joe trains ≥3×/wk for 4 wks AND still loves using it. KPI: workouts/week.

Phase 1 — Quiet share Next

  • Decide the name (Groove is crowded; revisit when public) open
  • Facing website + privacy policy page planned
  • Real PNG icons (192/512/maskable) planned
  • Ship free (no ads) to Google Play via PWABuilder ($25) planned
  • Hand to 5–10 friends
Gate → advance only if 3–4 friends still logging at 3 weeks.

Phase 2 — Templates & wedge Later

  • Templates — weekly schedule (Mon=Push, Tue=Jiu Jitsu, …) that auto-populates each day's moves; classes saved with their time box. Marquee Pro feature.
  • Personal records (best set per lift) + auto-prefill last working weight
  • Rest timer; year heatmap (Month/Year toggle in Calendar)
Gate → retained users ask to pay / invite others.

Phase 3 — Programs, AI & monetization Later

  • Programs — predefined goal-based plans (Weight Loss / Cardio / Muscle Growth) that progress over weeks + a current-status tracker
  • AI insights / analytics over training history (needs weeks of data first)
  • Apple App Store; Pro unlock via Supabase entitlement (KatchT comp/plan pattern) — no ads
Gate → decide: real product, or beloved niche tool. Only then weigh seriously vs KatchT.

Decisions log

  • Personal-first, product-optional, gated by evidence, behind the KatchT go-live gate. (Panel verdict, Jun 19)
  • Dedicated free Supabase project in a separate personal org → stays free after KatchT goes Pro.
  • Monetize via Pro unlock, not ads — ads earn pennies, need a heavier wrapper, and hurt the daily-user experience.
  • Logging model: reps/sets grid primary; tap-tally for sets; shorthand kept only for editing past days.
  • Name "Groove" is provisional — revisit at Phase 1 (public).
